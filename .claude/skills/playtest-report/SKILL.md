# playtest-report

Description: テストプレイ結果からバランス調整項目を整理し、実装計画をタスクリストとして生成する。
User-invocable: true

## Instructions

テストプレイのフィードバックを受け取り、バランス調整のタスクリストを生成する。

### 入力

ユーザーからテストプレイの結果・フィードバックを自然言語で受け取る。典型的な内容:
- 定数変更（例: 「恩寵ボーナスを5毎に3VPに変更」）
- ルール変更（例: 「ワーカー上限5人」）
- アップグレード調整（例: 「交易は1段階のみ」）
- 新機能追加（例: 「ピタリ賞に恩寵も付与」）

### 分析手順

1. **フィードバック分類**: 各項目を以下のカテゴリに分類
   - 定数変更: 値の修正のみで済むもの
   - ロジック変更: コードの処理フローに影響するもの
   - 新機能: 新しい仕組みの追加
   - UI変更: 表示・説明文の更新のみ

2. **影響範囲分析**: 各項目について影響を受けるファイル・関数を特定
   - main.py: CLI / GameEngine / Simulation のどのパスに影響するか
   - streamlit_app.py: import / UI説明文に影響するか
   - CLAUDE.md: 定数記述の更新が必要か

3. **タスクリスト生成**: TaskCreateを使い、以下の形式でタスクを作成
   - 各バランス調整項目を1タスクとして作成
   - 依存関係がある場合はaddBlockedByで設定
   - 最後に「GUIテストプレイ」「バランスシミュレーション実行」タスクを追加

4. **実装順序の提案**: 効率的な実装順序を提案
   - 定数変更 → ロジック変更 → 動作確認の順が基本
   - 互いに独立した変更は並列実施可能と明示

### 出力

タスクリスト生成後、ユーザーに以下を提示:
- 分類された変更項目一覧
- 各項目の影響範囲サマリー
- 推奨実装順序
- 確認後、`/game-rule-change` で各項目を適用可能であることを案内
