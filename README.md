# TsT (Coven)

トリックテイキングとワーカープレイスメントを組み合わせたカードゲームプロトタイプ。

## 必要環境

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (推奨)

## セットアップ

```bash
# リポジトリをクローン
git clone https://github.com/melon1891/coven.git
cd coven

# 依存関係をインストール (uv使用)
uv sync
```

## 起動方法

### CLIモード（対人プレイ）

```bash
# uv使用（推奨）
uv run python main.py

# または直接Python
python main.py
```

### 自動実行モード（Botのみ）

CI検証や動作確認用に4人全員Botで自動的にゲームが進みます。

```bash
# ランダムシード（毎回異なる結果）
uv run python main.py --auto

# 固定シード（再現可能）
uv run python main.py --auto --seed 12345
```

### Streamlit GUIモード

```bash
uv run streamlit run streamlit_app.py
```

## シミュレーション

ゲームバランス検証用の各種シミュレーションが可能です。

```bash
# カードランク別の勝率分析
uv run python main.py --simulate

# デッキ枚数別の分析
uv run python main.py --simulate-deck

# 負債ペナルティの分析
uv run python main.py --simulate-debt-penalty

# 恩寵ポイント閾値の分析
uv run python main.py --simulate-grace

# 金→恩寵変換レートの分析
uv run python main.py --simulate-gold-grace
```

## ゲームルール

### 基本情報

- **プレイ人数**: 4人（1人 + Bot 3人、または全員Bot）
- **ラウンド数**: 6ラウンド
- **勝利条件**: 最終ラウンド終了時に最も多くのVPを獲得

### カード構成

- **通常カード**: 4スート（♠♥♦♣）× ランク1-5 × 2デッキ = 80枚
- **切り札**: 2枚（ランクなし）

### ラウンドの流れ

1. **カード配布** - 各プレイヤーに5枚配布
2. **アップグレード公開** - 4枚のアップグレードカードを公開（R3は魔女）
3. **トリックテイキング** - 宣言 → 公開封印1枚 → 4トリック
4. **恩寵先行権** - 恩寵消費で同トリック数の相手より先に選択可能
5. **アップグレード選択** - トリック獲得順に選択（同数時は恩寵多い方優先、共有ボードに配置）
6. **ワーカープレイスメント** - ナショナルエコノミー方式で1人ずつ交互に配置
7. **給料支払い** - 初期ワーカーのみ給料支払い

### ワーカー配置（ナショナルエコノミー方式）

トリック勝数が多い順にプレイヤーが1ワーカーずつ交互に配置します。

#### 共通スポット（全体で1ワーカーのみ/ラウンド）

| スポット | 効果 |
|---------|------|
| Trade | 2金獲得 |
| Hunt | 1VP獲得 |
| Pray | 1恩寵獲得 |

#### オープンスポット（制限なし）

| スポット | 効果 |
|---------|------|
| Recruit | 2金消費 → ワーカー+1（次ラウンドから） |

#### 共有スポット（アップグレードカードから獲得、共有ボードに配置、同種最大2枚=Lv2）

全プレイヤーが使用可能。他プレイヤーのスポットを使用した場合、所有者は銀行から+1金を受け取ります。

| スポット | 獲得元 | Lv1効果 | Lv2効果 |
|---------|--------|---------|---------|
| Trade | UP_TRADE | 2金獲得 | 3金獲得 |
| Hunt | UP_HUNT | 1VP獲得 | 2VP獲得 |
| Pray | UP_PRAY | 1恩寵獲得 | 2恩寵獲得 |
| Ritual | UP_RITUAL | 2恩寵 or 2金（選択） | 3恩寵 or 3金（選択） |
| 交渉の魔女 | WITCH_NEGOTIATE | 1恩寵→2金 | - |

※ 同種のアップグレードカードを2枚取得するとLv2に昇格（効果+1）。魔女カードにレベルはありません。
※ 他プレイヤーが自分の共有スポットを使うと、所有者に+1金（銀行から）。

### 魔女カード（R3に登場）

7枚のプールから4枚が公開され、トリック獲得順に選択します。

| 魔女 | 種別 | 効果 |
|------|------|------|
| 黒路の魔女 | パッシブ | 交易スポット使用時+1金 |
| 血誓の討伐官 | パッシブ | 討伐スポット使用時+1VP |
| 群導の魔女 | パッシブ | 毎R給料-1 |
| 交渉の魔女 | 共有スポット | 1恩寵消費→2金獲得 |
| 祈祷の魔女 | パッシブ | 毎R+1恩寵 |
| 鏡の魔女 | パッシブ | 他プレイヤーの宣言成功時+1金 |
| 慎重な予言者 | パッシブ | 宣言0成功時、3恩寵/3金/2VPから選択 |

### 恩寵システム

ゲーム終了時、まず余った金貨が恩寵に変換され（2金=1恩寵）、その後恩寵がVPボーナスに変換されます（5恩寵毎→3VP）。

#### 恩寵獲得方法

| 方法 | 獲得量 | 条件 |
|------|--------|------|
| 共通Pray | 1恩寵 | 共通スポット使用 |
| 共有Pray | 1恩寵 | 共有スポット使用 |
| 共有Ritual | 2恩寵 or 2金 | 共有スポット使用（選択） |
| 宣言0成功 | +1恩寵 | 0宣言し、0トリック達成 |
| トリック0勝 | +1恩寵 | 宣言に関係なく0勝時 |
| 4位救済（選択時） | +2恩寵 | 2金の代わりに選択 |
| 《祈祷の魔女》 | +1恩寵/ラウンド | 魔女所持時、毎ラウンド終了時 |

#### 恩寵消費

- **手札全交換**: 封印前に3恩寵消費で手札5枚を全て引き直し
- **先行権**: 2恩寵消費で同トリック数の相手より先にアップグレード選択/ワーカー配置

## ファイル構成

```
coven/
├── main.py           # ゲームロジック・CLI
├── streamlit_app.py  # Streamlit GUI
├── pyproject.toml    # プロジェクト設定
├── game_log.jsonl    # ゲームログ出力先
├── README.md         # このファイル
└── CLAUDE.md         # Claude Code用ガイド
```

### カード入力形式（CLI）

`{スート}{ランク}` 形式:
- S = Spade, H = Heart, D = Diamond, C = Club, T = Trump
- 例: `S05`（スペード5）、`H03`（ハート3）、`T`（切り札）

### ログ出力

ゲームログは `game_log.jsonl` にJSONL形式で保存されます。
