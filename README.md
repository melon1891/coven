# TsT (Coven)

トリックテイキングとワーカープレイスメントを組み合わせたカードゲームプロトタイプ。

## 必要環境

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (推奨)

## セットアップ

```bash
# リポジトリをクローン
git clone https://github.com/melon1891/coven.git
cd coven

# 依存関係をインストール (uv使用)
uv sync
```

## 起動方法

### CLIモード（対人プレイ）

```bash
# uv使用（推奨）
uv run python main.py

# または直接Python
python main.py
```

### 自動実行モード（Botのみ）

CI検証や動作確認用に4人全員Botで自動的にゲームが進みます。

```bash
# ランダムシード（毎回異なる結果）
uv run python main.py --auto

# 固定シード（再現可能）
uv run python main.py --auto --seed 12345
```

### Streamlit GUIモード

```bash
uv run streamlit run streamlit_app.py
```

## シミュレーション

ゲームバランス検証用の各種シミュレーションが可能です。

```bash
# カードランク別の勝率分析
uv run python main.py --simulate

# デッキ枚数別の分析
uv run python main.py --simulate-deck

# 負債ペナルティの分析
uv run python main.py --simulate-debt-penalty

# 恩寵ポイント閾値の分析
uv run python main.py --simulate-grace
```

## ゲームルール

### 基本情報

- **プレイ人数**: 4人（1人 + Bot 3人、または全員Bot）
- **ラウンド数**: 6ラウンド
- **勝利条件**: 最終ラウンド終了時に最も多くの**VP（勝利点）**を獲得したプレイヤーが勝利
- **初期リソース**: 金貨5枚、ワーカー2人

### カード構成

- **通常カード**: 4スート（♠♥♦♣）× ランク1〜6 × 2デッキ = **96枚**
- **切り札（Trump）**: **4枚**（ランクなし、特殊カード）

### ラウンドの流れ

各ラウンドは以下の順序で進行します。

1. **カード配布** — 各プレイヤーに5枚配布
2. **アップグレード/魔女公開** — 5枚のアップグレードカードを公開（R3のみ魔女カード）
3. **宣言（ビッド）** — 獲得トリック数を0〜4で宣言
4. **手札交換**（任意）— 恩寵1消費で手札1枚をデッキトップと交換
5. **封印** — 手札から1枚を封印（残り4枚でプレイ）
6. **トリックテイキング** — 4トリックを実施
7. **宣言ボーナス判定** — 宣言通りなら+1VP
8. **恩寵ボーナス判定** — 宣言0成功で+1恩寵、トリック0勝で+1恩寵
9. **アップグレード選択** — トリック獲得数順に選択（金貨2枚を代わりに取ることも可）
10. **4位救済** — 最下位プレイヤーは金貨2枚 or 恩寵2点を選択
11. **ワーカープレイスメント** — ワーカーをアクションに配置
12. **給料支払い** — 初期ワーカーの給料を支払い（負債あればペナルティ）
13. **ワーカー登録** — 新規雇用ワーカーが次ラウンドから稼働
14. **魔女効果** — 《祈祷の魔女》所持者に+1恩寵

---

### トリックテイキング

#### 基本ルール

- **マストフォロー**: リードスートと同じスートの手札があれば必ず出す
- **フォローできない場合**: 切り札を含む任意のカードを出せる
- **切り札はリード不可**: トリックの最初に切り札を出すことはできない（手札が切り札のみの場合は例外）

#### トリックの勝者

1. **切り札が出された場合**: 最も早く切り札を出したプレイヤーが勝利
2. **切り札なしの場合**: リードスートの中で最もランクが高いカードが勝利
3. **同ランクの場合**: リーダー（先に出した方）が勝利

#### リーダーの決定

- 各ラウンドの最初のトリックはローテーションで決定
- 以降のトリックは前のトリックの勝者がリーダー

---

### 宣言（ビッド）

- トリック開始前に、獲得トリック数を**0〜4**の範囲で宣言
- 宣言通りのトリック数を獲得すると**+1VP**ボーナス
- 宣言を外してもペナルティなし
- **宣言0を達成した場合**: 追加で**+1恩寵**（《慎重な予言者》所持時は+2恩寵）

---

### アップグレードカード

トリック獲得数の多い順にアップグレードを選択できます。アップグレードの代わりに**金貨2枚**を受け取ることも可能です。

| アップグレード | 効果 | 最大レベル | 枚数 |
|--------------|------|-----------|------|
| 交易拠点改善 | TRADEの金貨獲得+2 | Lv2 | 6枚 |
| 魔物討伐改善 | HUNTのVP獲得+1 | Lv2 | 6枚 |
| 祈りの祭壇強化 | PRAYの恩寵獲得+1 | Lv2 | 2枚 |
| 見習い派遣 | 即座にワーカー1人雇用（2金消費、給料不要） | なし | 2枚 |
| 育成負担軽減 | そのラウンドの雇用数分、給料を割引 | Lv1 | 2枚 |
| 寄付の祭壇 | DONATEアクションを解放 | Lv1 | 2枚 |
| 儀式の祭壇 | RITUALアクションを解放 | Lv1 | 2枚 |

---

### 魔女カード

**R3（ラウンド3）**ではアップグレードの代わりに魔女カードが公開されます。魔女は取得後ゲーム終了まで永続的に効果を発揮します。

| 魔女 | 効果 |
|------|------|
| 《黒路の魔女》 | TRADEアクションの金貨+2 |
| 《血誓の討伐官》 | HUNTアクションのVP+1 |
| 《群導の魔女》 | ワーカー雇用時、給料コスト-1 |
| 《財宝変換の魔女》 | ゲーム終了時、残り金貨1枚につき恩寵1点に変換 |
| 《祈祷の魔女》 | 毎ラウンド終了時に恩寵+1 |
| 《的中の魔女》 | 宣言成功時に金貨+1 |
| 《慎重な予言者》 | 宣言0成功時の恩寵ボーナスが+2に強化 |

---

### ワーカープレイスメント

保有するワーカー数だけアクションを実行できます。

#### 基本アクション

| アクション | 効果（Lv0 → Lv1 → Lv2） |
|-----------|-------------------------|
| TRADE | 金貨獲得: 2 → 4 → 6 |
| HUNT | VP獲得: 1 → 2 → 3 |
| RECRUIT | ワーカー1人を雇用（2金消費、次ラウンドから稼働） |
| PRAY | 恩寵獲得: 1 → 2 → 3 |

#### アップグレードで解放されるアクション

| アクション | 解放条件 | 効果 |
|-----------|---------|------|
| DONATE | 寄付の祭壇 | 2金消費 → 恩寵1獲得 |
| RITUAL | 儀式の祭壇 | ワーカー1人 → 恩寵1獲得 |

#### ワーカーの種類

- **初期ワーカー**（2人）: 毎ラウンド給料を支払う必要がある
- **雇用ワーカー**（RECRUIT/見習い派遣で取得）: 給料不要

---

### 給料と負債

#### 給料カーブ

| ラウンド | R1 | R2 | R3 | R4 | R5 | R6 |
|---------|----|----|----|----|----|----|
| 給料/人 | 1金 | 1金 | 2金 | 2金 | 2金 | 3金 |

- **初期ワーカーのみ**が給料の対象（雇用ワーカーは不要）
- 「育成負担軽減」や「《群導の魔女》」で割引可能

#### 負債ペナルティ

- 給料を払えない場合、不足額×2のVPペナルティ
- 恩寵2点を消費して金貨1枚分の負債を軽減可能

---

### 恩寵（グレース）システム

恩寵ポイントは様々な方法で獲得・消費できるリソースです。

#### 恩寵の獲得方法

| 方法 | 獲得量 | 条件 |
|------|--------|------|
| PRAYアクション | 1〜3恩寵 | Lv0→1, Lv1→2, Lv2→3 |
| DONATEアクション | 1恩寵 | 2金消費（要アップグレード） |
| RITUALアクション | 1恩寵 | ワーカー1人消費（要アップグレード） |
| 宣言0成功 | +1恩寵 | 0を宣言し0トリック達成 |
| トリック0勝 | +1恩寵 | 宣言に関係なく0勝時 |
| 4位救済（選択時） | +2恩寵 | 金貨の代わりに選択 |
| 《祈祷の魔女》 | +1恩寵/ラウンド | 毎ラウンド終了時 |
| 《財宝変換の魔女》 | 金貨→恩寵 | ゲーム終了時のみ |

#### 恩寵の消費方法

| 消費アクション | コスト | 効果 | タイミング |
|--------------|--------|------|-----------|
| 手札交換 | 1恩寵 | 手札1枚をデッキトップと交換 | 封印前 |
| 後出し権 | 2恩寵 | そのトリックで最後にカードを出せる（ラウンド1回） | トリック中 |
| 負債軽減 | 2恩寵 | 金貨1枚分の負債を軽減 | 給料支払い時 |

#### 恩寵閾値ボーナス（ゲーム終了時、到達した最高閾値のみ適用）

| 恩寵ポイント | VPボーナス |
|-------------|-----------|
| 13点以上 | +8VP |
| 10点以上 | +5VP |

---

### 最終スコア計算

ゲーム終了時、以下の順序でスコアを計算します。

1. **ゲーム中に獲得したVP**（HUNT、宣言ボーナスなど）
2. **恩寵閾値ボーナス**: 恩寵10点以上で+5VP、13点以上で+8VP（最高のみ）
3. **金貨→VP変換**: 金貨2枚につき1VP
4. **負債ペナルティ**: 給料不足時に発生済み

同点の場合は金貨の多いプレイヤーが上位になります。

## ファイル構成

```
coven/
├── main.py           # ゲームロジック・CLI
├── streamlit_app.py  # Streamlit GUI
├── pyproject.toml    # プロジェクト設定
├── game_log.jsonl    # ゲームログ出力先
├── README.md         # このファイル
└── CLAUDE.md         # Claude Code用ガイド
```

### カード入力形式（CLI）

`{スート}{ランク}` 形式:
- S = Spade, H = Heart, D = Diamond, C = Club, T = Trump
- 例: `S06`（スペード6）、`H03`（ハート3）、`T`（切り札）

### ログ出力

ゲームログは `game_log.jsonl` にJSONL形式で保存されます。
