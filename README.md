# Coven（魔女協会）

トリックテイキングとワーカープレイスメントを組み合わせた4人用カードゲーム。

## セットアップ

### 必要環境

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (推奨)

### インストール

```bash
git clone https://github.com/melon1891/coven.git
cd coven
uv sync
```

### 起動方法

```bash
# CLIモード（1人 + Bot3人）
uv run python main.py

# 自動実行（Bot4人、動作確認用）
uv run python main.py --auto
```

## ゲームルール

### 概要

| 項目 | 内容 |
|------|------|
| プレイ人数 | 4人（1人+Bot3人 or 全員Bot） |
| ラウンド数 | 6ラウンド |
| 勝利条件 | ゲーム終了時、最もVP（勝利点）が多いプレイヤーの勝ち |
| 初期資源 | 金貨5枚、ワーカー2人 |

### カード構成

- **通常カード**: 4スート（♠♥♦♣）× ランク1〜5 × 2セット = **40枚**
- **切り札**: **2枚**（スートなし・ランクなし、最強）
- **合計**: **42枚**

---

## ラウンドの流れ

各ラウンドは以下の順で進行します。

**ターン順**: 各ラウンド、**恩寵が最も多いプレイヤー**がリード（同数時はラウンドロビン順）。宣言順・リード順ともにこの順で決まります。

**魔女候補**: ゲーム開始時にR3で登場する魔女候補5枚が全員に公開されます。

### 1. カード配布

各プレイヤーに**5枚**配布。

### 2. アップグレード公開

場に**5枚**のアップグレードカードを公開（R3のみ魔女カードが登場）。

### 3. トリックテイキング

#### 宣言

各プレイヤーは、このラウンドで何トリック取るかを**1〜4で宣言**します（《慎重な予言者》所持時のみ0宣言可能）。

- **手札交換（任意）**: 宣言後・封印前に**2恩寵**を消費して、手札から**任意の枚数**を選んで引き直せます。

#### 封印

手札5枚から**1枚を裏向きに封印**（使用不可に）。残り4枚でトリックを行います。

#### トリック（4回）

- リードプレイヤーが1枚出し、時計回りに1枚ずつ出す。
- **マストフォロー**: リードと同じスートがあれば必ず出す。なければ自由。
- **勝敗判定**: 切り札 ＞ リードスートの最大ランク。同ランクは先出し優先。

#### 宣言ボーナス

- **ピタリ賞**: 宣言通りのトリック数を取ったら **+1VP、+1恩寵**。
- **慎重な予言者**（魔女）所持時：宣言0成功で **3恩寵 / 3金 / 2VP** から選択。

### 4. アップグレード選択

トリック獲得数が多い順に、公開された5枚から1枚選択します。1枚は余ります。

- **同数時のタイブレーク**: 恩寵が多い方 ＞ 座席順

#### アップグレードカードの種類

| カード | 効果（共有スポット） |
|--------|---------------------|
| UP_TRADE | 共有交易スポットを獲得 |
| UP_HUNT | 共有討伐スポットを獲得 |
| UP_PRAY | 共有祈りスポットを獲得 |
| UP_RITUAL | 共有儀式スポットを獲得 |

選択したアップグレードカードは**共有ボード**に配置されます。誰でも使用でき、**他プレイヤーが使った場合は所有者に銀行から+1金**が入ります。

**レベルシステム**: 同種を2枚取得すると**Lv2**に昇格し、効果が+1されます（最大Lv2）。

### 5. ワーカープレイスメント

トリック獲得数が多い順に、**1ワーカーずつ交互**に配置します（ナショナルエコノミー方式）。全員のワーカーを使い切るまで繰り返します。

#### 共通スポット（全体で1人のみ/ラウンド）

| スポット | 効果 |
|---------|------|
| 交易 | 2金獲得 |
| 討伐 | 1VP獲得 |
| 祈り | 1恩寵獲得 |

誰かが使ったら、そのラウンドは他のプレイヤーは使えません。

#### 雇用スポット（制限なし）

| スポット | 効果 |
|---------|------|
| 雇用 | 2金消費 → ワーカー+1（次ラウンドから使用可、上限5人） |

何人でも使用可能。雇ったワーカーには給料が発生しません。

#### 共有スポット（共有ボード上）

アップグレードカードや魔女カードで獲得したスポット。**全プレイヤーが使用可能**です。

| スポット | Lv1効果 | Lv2効果 |
|---------|---------|---------|
| 共有交易 | 2金獲得 | 3金獲得 |
| 共有討伐 | 1VP獲得 | 2VP獲得 |
| 共有祈り | 1恩寵獲得 | 2恩寵獲得 |
| 共有儀式 | 2恩寵 or 2金（選択）| 3恩寵 or 3金（選択）|
| 交渉の魔女 | 1恩寵消費→2金獲得 | ―（レベルなし） |

**共有スポットのルール**:
- 各共有スポットには**1ラウンドにつき1ワーカー**のみ配置可能。
- **他プレイヤーのスポットを使用** → 所有者に銀行から**+1金**。
- **儀式の祭壇**はワーカーを**永久に1人消費**します（そのラウンドの給料は発生）。

### 6. 給料支払い

- **初期ワーカー**（最初の2人）のみ給料が発生します。雇用したワーカーには給料不要。
- 給料額はラウンドごとに変動: R1〜R4は**1金/人**、R5〜R6は**2金/人**。
- 金が不足すると**未払い分が負債として累積**。
- **恩寵による負債軽減**: 給料支払い前に2恩寵を消費して1金分の負債を回避可能。

### 7. 4位救済

そのラウンドでVPが最も低いプレイヤーは、**1金** or **2恩寵** のどちらかを獲得。

## 魔女カード（R3に登場）

ラウンド3では通常のアップグレードの代わりに魔女カードが登場します。8枚のプールから5枚が**ゲーム開始時に候補として公開**されます。

| 魔女 | 種別 | 効果 |
|------|------|------|
| 黒路の魔女 | パッシブ | 自分の交易スポット使用時 **+1金** |
| 血誓の討伐官 | パッシブ | 自分の討伐スポット使用時 **+1VP** |
| 群導の魔女 | パッシブ | 初期ワーカー **1人分の給料免除** |
| 交渉の魔女 | 共有スポット | 1恩寵消費 → **2金獲得**（共有ボードに配置） |
| 祈祷の魔女 | パッシブ | 祈りスポット使用時 **+1恩寵** |
| 鏡の魔女 | パッシブ | 他プレイヤーの宣言成功時 **+1金** |
| 慎重な予言者 | パッシブ | **0宣言を解禁**。成功時、**3恩寵 / 3金 / 2VP** から選択 |
| 魅了の魔女 | パッシブ | ゲーム終了時、所持ワーカー **1人=1VP** |

パッシブ魔女は取得者だけに効果があります。交渉の魔女のみ共有スポットとして共有ボードに配置されます。

## 恩寵（Grace）システム

恩寵は金貨・VPとは別の第三の資源です。

### 恩寵の獲得方法

| 方法 | 獲得量 | 条件 |
|------|--------|------|
| 共通Pray | 1恩寵 | 共通スポット使用 |
| 共有Pray | 1〜2恩寵 | 共有スポット使用（レベルによる） |
| 共有Ritual | 2〜3恩寵 | 共有スポット使用（金と選択、レベルによる） |
| ピタリ賞 | 1恩寵 | 宣言通りのトリック数達成 |
| 4位救済（選択時） | 2恩寵 | 1金の代わりに選択 |
| 祈祷の魔女 | 1恩寵/R | 魔女所持時、毎ラウンド終了時 |

### 恩寵の使い道

| 用途 | コスト | 効果 |
|------|--------|------|
| 手札交換 | 2恩寵 | 封印前に指定枚数を引き直し |
| 負債軽減 | 2恩寵 | 給料の未払い1金分を回避 |
| VP変換（終了時）| 5恩寵 | 3VP（自動、端数切捨て） |

---

## ゲーム終了時の得点計算

6ラウンド終了後、以下の順で最終得点を計算します。

1. **魅了の魔女VP**: 所持していればワーカー1人=1VP
2. **金 → 恩寵変換**: 余った金貨を恩寵に変換（**2金 = 1恩寵**）
3. **恩寵 → VP変換**: 恩寵をVPに変換（**5恩寵ごとに3VP**、端数切捨て）
4. **金 → VP変換**: まだ残っている金貨をVPに変換（**2金 = 1VP**、端数切捨て）
5. **負債ペナルティ**: 累積負債 × **-2VP**（上限なし）

最終VPが最も多いプレイヤーの勝利。同点時は金貨が多い方が上位。

## カード入力形式（CLI）

`{スート}{ランク}` 形式で入力します。

| 記号 | スート |
|------|--------|
| S | ♠ Spade |
| H | ♥ Heart |
| D | ♦ Diamond |
| C | ♣ Club |
| T | 切り札（ランクなし） |

例: `S05`（スペード5）、`H03`（ハート3）、`T`（切り札）

## シミュレーション

ゲームバランス検証用の各種シミュレーションが可能です。

```bash
uv run python main.py --simulate              # カードランク別勝率
uv run python main.py --simulate-deck          # デッキ枚数別分析
uv run python main.py --simulate-debt-penalty  # 負債ペナルティ分析
uv run python main.py --simulate-grace         # 恩寵ポイント分析
uv run python main.py --simulate-gold-grace    # 金→恩寵変換レート分析
```
